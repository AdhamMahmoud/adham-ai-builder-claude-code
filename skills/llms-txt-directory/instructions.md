```ts

## llms.txt Format

### Standard Structure
```
# Product Name

## Core Concepts
- Concept 1: Brief explanation
- Concept 2: Brief explanation

## API Reference
- Function/Component: Usage example

## Examples
```code
// Example 1
```

## Common Patterns
- Pattern 1: When to use
```

### Best Practices
- Concise, LLM-optimized content
- Code examples included
- Up-to-date with latest version
- Links to full docs

## Creating Your Own llms.txt

### Template
```markdown
# Your Product Name

Version: 1.0.0
Updated: 2024-01-01

## Overview
One-line description of what your product does.

```bash

```

import { Thing } from 'your-package'

```

## Why llms.txt?

### Benefits
- LLMs can reference latest docs
- Always up-to-date info
- Reduces hallucinations
- Standardized format
- Easy to maintain

### Use Cases
- AI coding assistants
- Documentation chatbots
- RAG systems
- Custom GPTs
- AI agents

## Verification

### Check if llms.txt Exists
```bash
curl https://[domain]/llms.txt
```

### Validate Format
- Plain text
- UTF-8 encoding
- Markdown formatting
- < 1MB size (for performance)
